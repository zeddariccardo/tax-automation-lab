Separar datos permanentes y datos anuales

Un expediente fiscal eficaz distingue la información que describe de forma estable al cliente — identidad, grupo, regímenes, opciones y controles — de la información correspondiente a un ejercicio concreto. Así se evita reconstruir cada año el mismo contexto y se hacen explícitas las actualizaciones necesarias.

Information Request List centralizada

La documentación solicitada no debe quedar repartida entre emails, notas y archivos separados. Una IRL centralizada vincula cada documento con área, estado, responsable, fecha límite, comentario e impacto en el trabajo. La finalización del expediente se vuelve medible y revisable.

Cálculo de impuestos y declaraciones

El working paper debe relacionar materialidad, cambios del negocio, operaciones extraordinarias, tax assets, fiscalidad diferida, transfer pricing, declaraciones y conciliaciones. Los elementos ausentes permanecen visibles como hallazgos o solicitudes abiertas.

Open points y conclusiones

Un open point es más que una nota. Debe indicar área, descripción, estado, impacto, respuesta recibida, eventual ajuste y conclusión. El modo Commentary sintetiza el trabajo; el modo Analítico conserva el detalle.

Archivo de clientes y traslado anual

El traslado al nuevo ejercicio debe ser controlado: los datos permanentes y las configuraciones pueden trasladarse, mientras que los importes, resultados y conclusiones del ejercicio anterior no deben copiarse automáticamente sin una decisión explícita.

Resultados, límites y responsabilidad

La herramienta produce Excel, PDF, JSON y reportes HTML y conserva los datos localmente en el navegador. No sustituye los procedimientos de auditoría, el juicio profesional ni la validación de las conclusiones fiscales.
