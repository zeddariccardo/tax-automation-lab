Security policy

Security

How the tools work, how to report problems and which limitations should be considered.

This is an informational translation. For legal interpretation, the Italian version prevails.

Local processing

The tools are designed to process files in the browser. They do not require an account, authentication or software installation and do not send uploaded documents to a website database.

Reporting

To report a vulnerability or unexpected behaviour, write to security@taxautomationlab.com. Include a description, steps to reproduce the issue and the observed impact. Do not send personal data or confidential documents.

Limitations

No system can be described as risk-free. The tools should be used with updated browsers, and outputs must be subject to professional review.
